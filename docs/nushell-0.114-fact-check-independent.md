# Nushell 0.114+ 独立事实核查（主源优先）

核查日期：2026-07-10
核查对象：`SKILL.md` 与 `references/*.md` 中面向 Nushell 0.114+ 的可验证技术陈述。被核查文件只用于提取 claims，不作为证据。
实测环境：Nushell MCP `0.114.0`，commit `57a036c4b011c027609b90afefb52f5930d84496`，默认实验选项包含 `enforce-runtime-annotations=true`。

## 结论摘要

- 0.114 的核心迁移说明整体准确：类型收紧、可选参数的 `nothing` 联合、显式子模块导出、`run`、SemVer、错误 `details`、表格导入 flags、大小写命令弃用等均有官方发布说明或源码支持。
- 安全章节存在三项需要立即修订的问题：`mktemp` 后直接 `save` 的示例不能工作；`safe-glob` 可被绝对路径绕过；不存在建议中的 `--no-expand` flag。
- 若干陈述方向正确但写得过度绝对：环境变量作用域、类型系统的安全收益、`do -c` 的“错误作为值”、`run --full-reparse` 的使用场景、外部命令 glob 展开均应补充边界。

## 建议的修订优先级

1. **P0：修复安全示例**：`mktemp` + `save --force`；重写 `safe-glob`；删除不存在的 `--no-expand`。
2. **P0：修复不可运行/不等价示例**：`o+e>| complete` 后不得读取 `stderr`；substring 使用半开区间；`rm -rf` 与 `test -f` 映射保持原语义；不要把 `source` 当成 parse-only 检查。
3. **P1：收紧安全模型措辞**：环境作用域、类型系统防注入、动态解析、typed glob 与 string 的差别，以及自动 CMD dispatch 与显式 `/C` 的边界。
4. **P1：澄清错误处理**：`do -c` 返回的是仍会传播的 error value；`$err.details` 是新增结构化字段但不是 error record 唯一字段。
5. **P2：降低绝对措辞**：diagnostic presentation、`run --full-reparse`、`to nuon --pretty`、streaming、`grid` 默认列、`par-each` 顺序和 Polars 性能阈值的当前实现细节。

## 可复现实测摘要（Nushell MCP 0.114.0）

```nu
# mktemp 创建了现有文件，普通 save 会失败
let tmp = (mktemp --suffix .tmp)
'data' | save $tmp
# => Destination file already exists

# base 目录并不限制绝对 glob
cd /tmp
glob '/etc/passwd' --depth 3
# => /etc/passwd

# string 外部参数不展开，glob typed value 会展开
let p = '*'; ^printf '<%s>\n' $p
let p = ('*' | into glob); ^printf '<%s>\n' $p

# 合流后 complete 没有 stderr 列
let merged = (^sh -c 'printf out; printf err >&2; exit 3' o+e>| complete)
$merged | columns
# => [stdout exit_code]

let separate = (^sh -c 'printf out; printf err >&2; exit 3' | complete)
$separate | columns
# => [stdout stderr exit_code]

# catch 的 0.114 error record
try { error make {msg: 'x'} } catch {|e|
  {top: ($e | columns), details: ($e.details | columns), json: $e.json?}
}
# top: [msg debug raw rendered details]
# details: [msg labels code url help inner]
# json: null
```
